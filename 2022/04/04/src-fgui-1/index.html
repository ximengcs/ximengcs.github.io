FGUI 源码剖析

发表于 2022-04-04 分类于 FGUI

概述

FGUI有编辑器端和平台SDK端，这里就分析UnitySDK。编辑器里可以将设计好的UI发布为描述文件和图集，描述文件里存储了标头，版本，UI布局等信息，之后在Unity端解析描述文件和图集(通过UIPackage加载)，通过FGUI自己的一套渲染流程，本质上就是封装了Unity的Mesh，通过MeshRender并且设置FGUI的shader,设置渲染数据来渲染UI, 然后使用者就可以调用FGUI提供的接口使用UI了。

目录结构

FGUI主要有两个大的目录

Resources中存放了渲染FGUI中文字和图片的shader。Scripts中当然是存档了FGUI的核心代码了。

Scrips

Core里面存放了FGUI的UI舞台Stage还有UI相机和渲染类，UI里面存放了和编辑器相对应的各种UI类，UI根节点GRoot，还有UIPackage。

UIPackage

核心类

EventListener
事件分发器，用来管理FGUI中所有UI的事件，onClick,OnTouch等，可以添加Event或者Capture，两种事件的区别就是Event会以冒泡的方式分发事件，也就是底->顶，而Capture则为顶->底。并且Event会在所有的Capture事件分发完成后才会开始分发

NTexture
表示了UI中的单个纹理资源，封装了Unity的纹理资源Texture，由于FGUI发布的是图集，一张大图里存放了多个纹理，所以FGUI用NTexture封装了单个纹理(实际上是提供了一些偏移信息和Rect，单个纹理的原始大小等关键信息)。

NGraphics
用来绘制UI的网格Mesh，所有的UI都需要用这个类来绘制，封装了Unity的Mesh, MeshFilter, MeshRender，通过标记Mesh为动态来处理各种动态修改：

并且在更新方法中调用了IMeshFactory的OnPopulateMesh方法:

Image，TextField都实现了OnPopulateMesh来生成自己的网格数据

DisplayObject
显示对象，封装了NGraphics，并且继承了EventListener，以及UI中的一些事件, 封装了大部分基础属性，如位置，缩放，大小，层等。DisplayObject下实现了一个DisplayObjectInfo，不过这个类的作用仅仅是在被销毁时设置一个销毁标志，还有在Inspector面板显示Info。 DisplayObject有一个生命周期Update方法，每次调用时会渲染一帧:

所以要渲染出对象，就需要上次实现来调用这个Update

Container
Container管理了一个显示列表作为一个DisplayObject的容器，可以添加或移除显示对象DisplayObject(作为它的Child)，它还实现了渲染模式(相机空间，世界空间)，以及UI的点击检测。
它也继承了DisplayObject，并且重写了Update方法，会调用存储的孩子对象的Update。所以它也需要上层来调用这个Update。

Stage
Stage管理了所有UI对象的更新，点击检测，声音播放，核心事件。Stage继承了Container，并且重写了Update方法，调用所有在Stage中的显示对象DisplayObject。而Stage的Update由StageEngine来每帧调用，StageEngine核心代码:

StageEngine是一个MonoBehavior，可以看到Stage UI是在LateUpdate中更新的。

GRoot实际上就是管理通过Container管理了所有的UI，然后实现了显示窗口和PopupUI的一些方法，窗口实际上还是通过自己的Container添加，而PopupUI实现了单独的管理。

GObject
GObject继承了EventListener事件分发器，封装了显示对象Displayobject，PackageItem和所有的UI基本属性，所有的基本UI事件。

GComponent
GComponent继承了GObject，额外封装了Container，并且管理了一个GObject列表，管理了其中所有的孩子, 包括添加移除等，添加时它会将待添加元素GObject的显示对象Di是playObject添加到自己的Container中:

而GRoot也是继承了GComponent，来管理它的孩子对象。

GImage, GTextField, GList…
理解以上的类后，这些UI实现类都是通过继承或封装GObject， GComponent来实现自己的基本功能。

其他类

Core/HitTest

存放了关于点击检测的类，实现了IHitTest接口可以检测是否点击到区域，Container中管理了一个HitArea用来检测点击。
还有各种实现如ColliderHitTest碰撞体点击实现，MeshColliderHitTestMesh碰撞体检测，RectHitTest区域检测等。

Core/Mesh

实现了各种类型的Mesh数据生成，如FillMesh填充，LineMesh线等。IMeshFactory和VertexBuffer也在这里。VertexBuffer就是顶点数据的缓存，IMeshFactory的接口OnPopulateMesh被这里的各种实现类实现，它会在绘制更新中被调用(如NGraphics的Update中)。

Core/Stats

FGUI运行时的各种统计信息

Event

FGUI的UI事件实现：
EventDispatcher, 管理了一个EventBridge，用来分发各种UI事件和Capture，事件的派发通过EventBridge
EventBridge，可以添加移除事件监听和Capture
EventListener, 事件监听器，封装了EventBridge，触发事件时通过Bridge来调用EventDispatcher的分发事件方法

EventContext, 事件上下文，派发事件的上下文数据, 监听者收到事件时可以解析里面的数据来处理, 包含了InputEvent数据。
InputEvent 封装了Unity中的输入数据，如KeyCode等。

Filter

IFilter 这个接口有Update和Dispose方法，和一个Displayobject的属性，Update方法会在Displayobject的更新法中最后阶段被调用，可以实现各种效果, 这里的BlurFilter和ColorFilter就是两个实现

Gesture

操作手势相关的实现类，继承了EventDispatcher来分派手势相关的事件

Tween

实现了UI的补间方法

UI

对应于编辑器中的各种UI实现类，GComponent, GButton, GList, GLoader …, 基类GObject

Utils

各种工具类，UBB的处理，XML处理，Zip处理，HTML处理，计时器Timer …
