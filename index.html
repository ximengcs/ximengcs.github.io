FGUI 源码剖析

发表于 2022-04-04 分类于 FGUI

概述

FGUI有编辑器端和平台SDK端，这里就分析UnitySDK。编辑器里可以将设计好的UI发布为描述文件和图集，描述文件里存储了标头，版本，UI布局等信息，之后在Unity端解析描述文件和图集(通过UIPackage加载)，通过FGUI自己的一套渲染流程，本质上就是封装了Unity的Mesh，通过MeshRender并且设置FGUI的shader,设置渲染数据来渲染UI, 然后使用者就可以调用FGUI提供的接口使用UI了。

目录结构

FGUI主要有两个大的目录

Resources中存放了渲染FGUI中文字和图片的shader。Scripts中当然是存档了FGUI的核心代码了。

Scrips

Core里面存放了FGUI的UI舞台Stage还有UI相机和渲染类，UI里面存放了和编辑器相对应的各种UI类，UI根节点GRoot，还有UIPackage。

UIPackage

UIPackage的主要作用有两个，其一是加载编辑器中发布的描述文件和图集，还有就是创建出UI类。

DirectX 数学库 - 矩阵

发表于 2021-12-03 分类于 DirectX

矩阵

介绍

DirectX中XMMATRIX表示4*4类型的矩阵，XMMATRIX由4个XMVECTOR向量实例所组成，以次来发挥SIMD技术，但是在32位Windows系统上不支持，因为32位Windows上只通过寄存器传递前3个XMVECTOR参数，其余在堆栈上。

头文件

和向量一样都定义在DirectXMath.h中

注意点

传递参数时1个XMMATRIX算作4个XMVECTOR参数

传递参数规则：传递参数时如果XMMATRIX不超过两个，第一个作为FXMMATRIX类型传递，其余的作为CXMMATRIX类型传递，其余规则和向量规则相同。

DirectX 数学库 - 向量

发表于 2021-11-28 更新于 2021-11-29 分类于 DirectX

向量

介绍

DirectXMath是DirectX专用数学库，是Windows SDK的一部分。

采用了128位宽的单指令多数据流(SSE2)寄存器来处理数学运算，所以一条SSE2指定就可以同时处理4个32位的浮点数或整数。

头文件

DirectXMath头文件：DirectXMath.h

DirectXMath所需的数据类型：DirectXPackedVector.h

文件中的代码都存在于DirectX命名空间中。

配置

x86平台：需要启用SSE2指令集

Project Properties -> Configuration Properties -> C/C++ -> Code Generation -> Enable Enhanced Instruction Set

x64平台：无需额外操作，所有x64平台CPU都对SSE2指令集支持
